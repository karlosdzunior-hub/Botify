1. Общее описание проекта
Bot Factory — это Telegram-бот, который позволяет любому пользователю создать собственного Telegram-бота без знания программирования.
Пользователь описывает идею → ИИ задаёт вопросы → показывает план → генерирует код → проверяет ошибки → запускает бота на сервере.
 
1.1 Цели проекта
• Создать SaaS-платформу для автоматической генерации Telegram-ботов
• Монетизировать через систему кредитов + подписки
• Обеспечить автономную работу: генерация, проверка и запуск без участия оператора
• Выйти на окупаемость при 50+ платящих пользователях
 
1.2 Целевая аудитория
• Малый бизнес (кафе, салоны, магазины) — боты для записи и каталогов
• Контент-мейкеры — боты для рассылок и работы с аудиторией
• Фрилансеры — быстрое создание прототипов для клиентов
• Агентства — White Label решение под своим брендом
 

2. Функциональные требования
2.1 Главный бот — пользовательский флоу
Шаг 1 — Приём идеи
• Пользователь отправляет текстовое описание желаемого бота
• Бот отвечает: принял, анализирую...
• ИИ (Groq Llama) классифицирует тип бота: простой / сложный / мини-апп
 
Шаг 2 — Уточняющие вопросы
• ИИ задаёт 3–4 конкретных вопроса по идее
• Вопросы отправляются одним сообщением, пользователь отвечает свободным текстом
• Бот запоминает контекст через FSM (конечный автомат состояний)
 
Шаг 3 — Превью и подтверждение
• ИИ составляет план: список функций, команды, сценарии использования
• Показывает расчётную стоимость в кредитах
• Кнопки: [✅ Создать] [✏️ Изменить план] [❌ Отмена]
 
Шаг 4 — Генерация кода
• Запрос токена бота у пользователя (через @BotFather)
• ИИ (Groq Qwen Coder) генерирует полный код на Python + aiogram 3
• Прогресс-бар в сообщении: Генерирую... Проверяю... Запускаю...
 
Шаг 5 — Авто-проверка и исправление ошибок
• Код запускается в Docker sandbox с ограниченными ресурсами
• При синтаксической ошибке — ИИ (Gemini Flash) автоматически исправляет
• До 3 итераций исправлений. Если не вышло — уведомление + возврат кредитов
 
Шаг 6 — Запуск и уведомление
• Бот запускается как subprocess на сервере
• Пользователь получает: ссылку на бота, статус, кнопки управления
• Кнопки: [🔄 Перезапустить] [⏹ Остановить] [📋 Логи] [📥 Скачать код]
 
2.2 Управление ботами (/mybots)
• Список всех ботов пользователя с иконкой статуса (✅/⛔)
• Команды: /mybots, /stop, /restart, /logs, /status
• Просмотр последних 20 строк логов прямо в чате
• Удаление бота с подтверждением — освобождает ресурсы сервера
 
2.3 Редактирование существующего бота
• Пользователь описывает что изменить текстом
• ИИ вносит изменения в существующий код (не генерирует заново)
• Стоимость: 8 кредитов за добавление функции
• Бот автоматически перезапускается после изменений
 
2.4 Мини-апп — ChatGPT-подобный интерфейс
Основной интерфейс пользователя — Telegram Mini App с чатом как в ChatGPT. Всё общение с ИИ происходит через него, а не через обычный чат бота.
 
Первый экран (главный)
• Шапка: кнопка ☰ (3 полоски) слева, название «Bot Factory» по центру, баланс кредитов справа
• Основная область: чат с ИИ — сообщения пользователя справа, ответы ИИ слева
• При первом входе: приветствие + подсказка что написать
• Поле ввода снизу с кнопкой отправки
 
Блок прогресса генерации (в чате)
🤔 Анализирую запрос     ✅
  Составляю план        ✅
  Запрашиваю токен      ✅
  Генерирую код         ⏳
  Проверяю ошибки       ⏸️
 
⏱ 00:23   💳 -18 кредитов
 
• Шаги обновляются в реальном времени через WebSocket
• Таймер показывает сколько времени идёт генерация
• Счётчик кредитов списывается по мере выполнения шагов
 
Запрос токена бота
• После подтверждения плана — в чате появляется сообщение с инструкцией
• Пользователь получает токен через @BotFather и вставляет в поле ввода
• Токен валидируется через Telegram API перед генерацией
• Токен хранится зашифрованным (Fernet AES-128)

3.3 Схема отказоустойчивости
• Groq недоступен → автоматический переключатель на Gemini Flash
• Gemini недоступен → повторная попытка через Groq Qwen
• Claude недоступен (только для мини-аппов) → Gemini 2.0 Pro как фолбэк
• Все переключения логируются, пользователь не замечает сбоя
• Повторные попытки (retry) с экспоненциальной задержкой: 1с, 2с, 4с

5. Безопасность и ограничения
5.1 Изоляция созданных ботов
• Каждый бот запускается в отдельном subprocess со своей директорией
• Нет доступа к файлам других пользователей
• Лимит ресурсов: 128MB RAM, 0.5 vCPU на бота
• Лимит на количество одновременных запросов к Telegram API
 
5.2 Проверка кода перед запуском (Sandbox)
• Статический анализ: ast.parse() — синтаксическая проверка
• Docker sandbox: запуск с флагом --network=none (нет интернета)
• Запрещённые паттерны в коде: os.system, subprocess, __import__, open('/etc')
• Сканер на вредоносный код через ИИ-промпт перед генерацией
 
5.3 Фильтрация запросов пользователей
Запрещено создавать ботов для:
• Массовых рассылок без согласия получателей (спам)
• Сбора персональных данных без уведомления
• Мошенничества, фишинга, скама
• Нарушения ToS Telegram
Фильтрация: системный промпт + список ключевых слов + ИИ-модерация запроса.

6.3 Административная панель (/admin)
Доступна только владельцу (ADMIN_USER_ID в .env). Команды:
• /admin stats — общая статистика: юзеры, боты, генерации, доход
• /admin users — список пользователей с балансом и тарифом
• /admin ban <user_id> — заблокировать пользователя
• /admin bots — список всех активных ботов на сервере
• /admin logs <user_id> — логи конкретного пользователя
• /admin broadcast — рассылка всем пользователям
• /admin credits <user_id> <amount> — начислить кредиты вручную

10. Нефункциональные требования
10.1 Производительность
• Время ответа бота на команду: не более 2 секунд
• Время генерации простого бота: 30–60 секунд
• Время генерации сложного бота: 60–120 секунд
• Одновременных генераций: до 10 (очередь для остальных)
 
10.2 Надёжность
• Uptime главного бота: 99.5%+
• Автоперезапуск при падении (supervisor watchdog)
• Ежедневный бэкап базы данных
• Логирование всех ошибок в отдельный файл + уведомление администратору
 
10.3 Безопасность
• Токены ботов хранятся зашифрованными (Fernet AES-128)
• Все API-ключи только в .env, никогда в коде
• Ограничение доступа к директориям других пользователей (chmod 700)
• HTTPS для будущего веб-интерфейса
 
10.4 Масштабируемость
• Архитектура позволяет горизонтальное масштабирование (несколько серверов)
• Celery workers можно запускать на отдельных машинах
• PostgreSQL можно вынести на отдельный сервер без изменения кода
